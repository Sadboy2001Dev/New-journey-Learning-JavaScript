<h1>Exceptions handling and Error handling</h1>

<p>The first step to writing robust, high-quality software is acknowledging that it will have errors. The second step is anticipating those errors and handling them in a reasonable fashion.</p>

<p><strong>Exception handling</strong> is a mechanism that came about to deal with errors in a controlled fashion. It’s called exception handling because it’s meant to deal with exceptional circumstances—that is, not the errors you anticipate, but the ones you don’t.</p>

<p>An application that is designed to be used by the general, untrained public may anticipate a lot more unpredictable behavior than an application designed to be used by trained users.</p>

<p>In this chapter we're going to learn how to deal with exceptions and error.</p>
